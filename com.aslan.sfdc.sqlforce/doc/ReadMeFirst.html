SQLForce -- A Command Line Tool for Working with SalesForce

SQLForce was born out of the frustration of not having basic ANSI SQL commands for manipulating SalesForce databases. In short, SQLForce provide ANSI like SOQL commands that can save you a lot of time when maintaining SalesForce.

To use SQLForce you will need:

  • A Java 1.6 (or later) JRE. If you are looking at this project I am sure you already know that you can pick this up at java.sun.com
  • Your SalesForce credentials plus a generated security token.

The simplest way to launch SQLForce is:

  • java -jar sqlforce.jar
This will launch SQLForce without connecting to SalesForce. Once in, type HELPto discover the available commands. Hint: The command to connect to SalesForce is CONNECT.

SQLForce also has a handful of command line switches like any other program. To discover them type:

  • java -jar sqlforce.jar -#
One of the first things you will want to do is put your Salesforce credentials into a configuration file so you can avoid typing: 
CONNECT PRODUCTION gsmithfarmer@gmail.com MyPassword MySecurityToken
everytime you want to connect to Salesforce. A configuration file keeps your credentials out of scripts and supports the shorter form of the CONNECT command: 
CONNECT PROFILE profileName
To learn about the SQLForce configuration file see the Credential Setup documentation.

Another task you may want to do is package the "java -jar sqlforce.jar" into a script file (.bat, .sh, .csh, etc.). I usually do this and also include login credentials or a profile reference in my script. Examples: 

java sqlforce.jar -connect myProfileName -logINFO

OR

java sqlforce.jar -connect "PRODUCTION,gsmithfarmer@gmail.com,MyPassword,MySecurityToken" -log INFO

The first step you should take is to read some of the use cases. They included descriptions of problems that have been solved with SQLForce.
